### GitHub CLI (`gh` 명령어) 설치 및 사용법

`gh`는 터미널(명령 프롬프트, PowerShell, Git Bash 등)에서 GitHub의 기능(로그인, 저장소 생성, PR 관리 등)을 직접 다룰 수 있도록 돕는 GitHub 공식 CLI 도구입니다.

#### 1. GitHub CLI 설치하기 (Windows)
Windows 터미널(PowerShell 또는 cmd)에서 다음 명령어로 간편하게 설치할 수 있습니다.
```bash
winget install --id GitHub.cli
```
* 또는 공식 홈페이지(https://cli.github.com/)에서 설치 파일을 다운로드하여 설치할 수 있습니다.
* 설치 완료 후 터미널을 재실행하여 `gh --version`으로 정상 설치 여부를 확인합니다.

#### 2. 인증(로그인)하기
GitHub 계정을 CLI에 등록하고 인증합니다.
```bash
gh auth login
```
* **대화형 선택 옵션 추천 흐름**:
  1. What account do you want to log into? -> **GitHub.com** 선택
  2. What is your preferred protocol for Git operations? -> **HTTPS** (또는 SSH) 선택
  3. Authenticate Git with your GitHub credentials? -> **Yes** 선택
  4. How would you like to authenticate GitHub CLI? -> **Login with a web browser** 선택
  5. 화면에 나온 일회용 인증 코드(One-time code)를 복사한 뒤 Enter를 누르고, 열리는 브라우저 창에 붙여넣어 인증을 완료합니다.

#### 3. 상태 확인하기
현재 어떤 계정으로 로그인되어 있고, 토큰이 유효한지 확인합니다.
```bash
gh auth status
```

#### 4. 여러 계정 간 스위칭(전환)하기
GitHub CLI는 한 번 로그인해둔 계정 정보를 기억하므로, 아래 명령어로 간편하게 계정을 바꿀 수 있습니다.
```bash
gh auth switch
```
* 명령어를 입력한 뒤, 화살표 키를 사용하여 전환하고 싶은 계정을 선택하고 Enter를 누르면 즉시 해당 계정으로 전환됩니다.

#### 5. 로그아웃하기
더 이상 사용하지 않는 계정의 인증 정보를 삭제합니다.
```bash
gh auth logout
```
* 로그아웃할 계정(GitHub.com 등)을 선택하여 인증 정보를 제거합니다.
